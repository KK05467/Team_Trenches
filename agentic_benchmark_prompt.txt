================================================================================
            APPENDIX A: ASML HIGH-NA EUV LITHOGRAPHY TECHNICAL DATASHEETS
================================================================================
Lithography resolution is governed by the Rayleigh criterion:
  CD = k1 * (lambda / NA)
Where:
  - CD: Critical Dimension (minimum printable feature size).
  - k1: Process factor (physical limit for single-exposure projection is 0.25).
  - lambda: Wavelength of light source (13.5 nm for Extreme Ultraviolet lithography).
  - NA: Numerical Aperture of the optical lens system.

Low-NA EUV systems (such as NXE:3400C, NXE:3600D, and NXE:3800E) operate at NA = 0.33. 
Applying the Rayleigh formula:
  CD_low = 0.25 * (13.5 / 0.33) ≈ 10.2 nm minimum half-pitch resolution.
High-NA EUV systems (EXE:5000, EXE:5200) increase NA to 0.55.
Applying the Rayleigh formula:
  CD_high = 0.25 * (13.5 / 0.55) ≈ 6.1 nm minimum half-pitch resolution.

Anamorphic Optical Reduction System:
In traditional 0.33 NA systems, the magnification reduction factor is 4x symmetric (both X and Y directions). Increasing the NA to 0.55 requires the light to hit the mask at larger angles. If a standard 4x reduction mask were used, the light rays would overlap on the mask, causing severe shadow effects.
To solve this, ASML and Zeiss developed anamorphic optics:
- X-axis magnification: 4x reduction.
- Y-axis magnification: 8x reduction.
Because the reduction along Y is doubled (8x), the exposed field size on the wafer is cut in half.
- Standard 0.33 NA reticle field size: 26 mm x 33 mm (858 mm²).
- High-NA 0.55 NA reticle field size: 26 mm x 16.5 mm (429 mm²).
This is known as a half-field exposure. As a result, designs larger than 429 mm² must be printed using stitching (exposing the top half and bottom half separately and merging them at the boundary). Wafer stitching reduces overall throughput, exposes designs to overlay alignment errors at the boundary interface, and limits maximum monolith chip size.

Wafer Throughput Specifications:
- NXE:3600D (0.33 NA): ~160 wafers per hour (wph) at 30 mJ/cm² dose.
- EXE:5000 (0.55 NA): ~150 wafers per hour.
- EXE:5200 (0.55 NA): ~220 wafers per hour, utilizing advanced high-speed wafer stages and higher-power EUV source collectors (exceeding 500W to compensate for higher optical losses in anamorphic lens arrays).

Wafer Fabrication Reticle Limits for Blackwell (TSMC 4NP):
TSMC 4NP is a highly optimized custom variant of the 5nm family. At these nodes, low-NA EUV requires multi-patterning (double or triple exposure) for critical metal layers, increasing mask counts, overlay alignment tolerances, and defect density risks. 
The standard physical reticle limit for DUV and low-NA EUV scanner stages is 858 mm² (26mm x 33mm).
The Blackwell B200 die:
- Consists of two fully reticle-limited monolithic dies fabricated on TSMC 4NP, each roughly 800 mm² in size.
- Total combined silicon size under the package exceeds 1600 mm².
- The dies are linked together via a 10 TB/s NVLink Chip-on-Wafer-on-Substrate (CoWoS-L) passive silicon interposer with active link blocks.
Because a single die cannot physically exceed 858 mm² due to scanner reticle mask dimensions, Blackwell B200 must rely on two separate dies stitched on a packaging substrate rather than a single monolithic chip.

================================================================================
       APPENDIX B: CUDA GPU SIMULATION: TILED MULTIPLICATION & BARRIERS
================================================================================
In a real GPU (NVIDIA Streaming Multiprocessor), global memory bandwidth is a major bottleneck. Doing a naive matrix multiplication:
  C[i][j] = sum(A[i][k] * B[k][j])
Requires loading elements from global memory multiple times, leading to high latency and memory traffic.

GPU Tiled Matrix Multiplication optimizes memory access by using Shared Memory:
1. Matrix A and B are split into tiles of dimension BLOCK_SIZE x BLOCK_SIZE (e.g., 16x16 threads).
2. Each thread in a CUDA thread block is mapped to local indices:
     tx = threadIdx.x
     ty = threadIdx.y
   And global indices:
     row = blockIdx.y * blockDim.y + threadIdx.y
     col = blockIdx.x * blockDim.x + threadIdx.x
3. Allocate shared memory arrays:
     __shared__ float s_A[BLOCK_SIZE][BLOCK_SIZE];
     __shared__ float s_B[BLOCK_SIZE][BLOCK_SIZE];
4. Loop through the matrices in steps of BLOCK_SIZE:
   a. Each thread loads exactly one element of Matrix A and one element of Matrix B from global memory into the shared memory tile:
        s_A[ty][tx] = A[row][m * BLOCK_SIZE + tx]
        s_B[ty][tx] = B[m * BLOCK_SIZE + ty][col]
   b. Synchronize threads in the block to ensure the entire tile is loaded:
        __syncthreads()
   c. Accumulate the partial dot product:
        for k from 0 to BLOCK_SIZE:
            sum += s_A[ty][k] * s_B[k][tx]
   d. Synchronize threads again to ensure computation is done before the next tile overwrites the shared memory buffer:
        __syncthreads()
5. Once all tiles have been processed, the thread writes its accumulated sum to the global memory matrix C:
     C[row][col] = sum

Boundary Handling:
If the matrix size (M, K, N) is not a multiple of BLOCK_SIZE, boundary checks must protect against memory access violations:
  - If row < M and (m * BLOCK_SIZE + tx) < K: load A, else load 0.0.
  - If (m * BLOCK_SIZE + ty) < K and col < N: load B, else load 0.0.
  - Write to C only if row < M and col < N.
